## InfluxDB 조회 구조

measurement는 주기별로 분리되어 있다 (`candle_1m`, `candle_1h`, `candle_4h`, `candle_1d`, `candle_1w`, `candle_1M`).
interval 파라미터를 measurement 이름으로 매핑하여 쿼리한다.

| interval | measurement |
|----------|-------------|
| `1m` | `candle_1m` |
| `1h` | `candle_1h` |
| `4h` | `candle_4h` |
| `1d` | `candle_1d` |
| `1w` | `candle_1w` |
| `1M` | `candle_1M` |


## 커시 기반 페이징 적용

차트를 과거 방향으로 넘기면 과거 캔들을 조회해 이어 붙여야 한다. 이때 캔들을 오프셋 대신 커서 기반으로 페이징 조회한다.

## 진행 중 캔들 즉석 집계

커서가 없는 최신 조회에서만, 아직 닫히지 않은 현재 구간의 캔들을 즉석 집계해 응답 맨 끝에 덧붙인다. 과거 스크롤(커서 있음)에는 덧붙이지 않는다.

### 시간 경계 계산

`CandleWindow`(도메인) 가 현재 시각과 타임존으로 현재 구간 시작(`periodStart`)과 하위 경계(일·시·분 시작)를 계산한다. 일·주·월·4시간 경계는 타임존에 따라 갈리므로 타임존을 주입받아 자른다. 거래소별 기준은 spec 의 *일 경계* 표를 따른다.

빗썸 KST 경계는 저장된 빗썸 상위봉이 KST 로 재적재되어 있어야 진행봉과 어긋나지 않는다. 그 재적재 전까지 어댑터는 전 거래소에 UTC 를 넘겨 저장된 값과 자기일관을 유지한다.

### 하위봉 조합

현재 구간을 완성된 부분과 진행 중인 부분으로 나누어, 완성된 부분은 저장된 상위봉으로 채우고 남은 조각만 더 잘게 채운다. 어떤 주기든 한 조각의 조회량이 작게 유지된다.

| 조각 | measurement | 범위 |
|------|-------------|------|
| 완성된 날 | `candle_1d` | 구간 시작 ~ 오늘 0시 |
| 완성된 시간 | `candle_1h` | 오늘 0시 ~ 현재 시 시작 |
| 완성된 분 | `candle_1m` | 현재 시 시작 ~ 현재 분 시작 |
| 진행 중인 분 | `ticker_raw` | 현재 분 시작 ~ 현재 |

- 일·4시간은 완성된 부분을 1시간봉으로 채운다. 하위 경계가 모두 정시라 거래소 일 경계가 무엇이든 빈틈 없이 맞물린다.
- 각 조각을 시가(first)·고가(max)·저가(min)·종가(last)로 집계한 뒤, 시가는 첫 조각, 종가는 마지막 조각, 고가·저가는 전체에서 취해 하나로 합친다.
- OHLC 집계는 결합법칙을 만족하므로(고가의 고가는 전체 고가) 상위봉과 하위봉을 섞어 합쳐도 손실이 없다.
- 해당 구간에 데이터가 없으면 진행봉을 덧붙이지 않는다.

### 마감 직후 중복 방지

구간이 막 닫혀 집계 Task 가 이미 같은 시각의 봉을 써 둔 경우, 마지막 닫힌 봉의 시각이 진행봉과 같으면 덧붙이지 않고 진행봉으로 교체한다.

## API 명세

`GET /api/candles?exchange={exchange}&coin={coin}&interval={interval}&limit={limit}&cursor={cursor}`

### Query Parameters

| 필드 | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| exchange | String | O | | 거래소 코드 (`UPBIT`, `BITHUMB`, `BINANCE`) |
| coin | String | O | | 코인 심볼 (`BTC`, `ETH` 등) |
| interval | String | O | | 캔들 주기 (`1m`, `1h`, `4h`, `1d`, `1w`, `1M`) |
| limit | Integer | X | 60 | 조회할 캔들 개수 (1~200) |
| cursor | String | X | | ISO 8601 타임스탬프. 이 시각 이전의 캔들을 조회한다 (과거 스크롤용) |

### Response

```json
{
  "status": 200,
  "code": "SUCCESS",
  "message": "캔들 데이터를 조회했습니다.",
  "data": [
    {
      "time": "2026-03-10T00:00:00Z",
      "open": 68500000.0,
      "high": 69200000.0,
      "low": 67800000.0,
      "close": 68900000.0
    },
    {
      "time": "2026-03-11T00:00:00Z",
      "open": 68900000.0,
      "high": 70100000.0,
      "low": 68400000.0,
      "close": 69750000.0
    }
  ]
}
```

### 응답 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| time | String | 캔들 시작 시각 (ISO 8601) |
| open | Double | 시가 |
| high | Double | 고가 |
| low | Double | 저가 |
| close | Double | 종가 |

### 에러 응답

| code | status | 설명 |
|------|--------|------|
| INVALID_CANDLE_INTERVAL | 400 | 지원하지 않는 캔들 주기 |
| INVALID_CANDLE_LIMIT | 400 | limit 범위 초과 (1~200) |
