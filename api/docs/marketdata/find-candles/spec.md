## 목적

캔들(OHLC) 차트 데이터를 조회한다. 시계열 캔들 데이터를 커서 기반 페이징으로 제공한다.

- 거래소별, 코인별 캔들 차트를 제공하기 위한 기능이다

### 사용처

- 마켓 탭에서 거래소·코인을 선택하면 캔들 차트를 표시한다

## 조회 항목

- 거래소·코인별 캔들(OHLC) 시계열 데이터: 각 캔들의 시가·고가·저가·종가와 시각
- 6개 주기(1분, 1시간, 4시간, 일, 주, 월)의 캔들
- 커서 기반 페이징으로 과거 캔들을 스크롤하며 조회
- 최신 조회 시 아직 닫히지 않은 현재 구간의 캔들을 실시간으로 집계해 최신 봉으로 포함

## 진행 중 캔들

집계 Task 는 구간이 끝난 뒤에만 상위봉(1시간·4시간·일·주·월)을 기록하므로, 진행 중인 현재 구간의 봉은 저장소에 존재하지 않는다. 이 봉을 서버가 내려주지 않으면 클라이언트는 접속 이후 수신한 체결만으로 봉을 그려, 시가가 실제 구간 시작가가 아니라 "접속 시점 가격"이 되어 몸통이 왜곡된다.

이를 막기 위해 커서가 없는 최신 조회에서는 현재 구간의 캔들을 즉석 집계하여 최신 봉으로 포함한다.

- 시가·고가·저가는 구간 시작부터 현재까지의 실제 체결을 반영한다.
- 종가는 가장 최근 체결가로, 클라이언트가 실시간 체결로 계속 갱신한다.
- 커서가 있는 과거 조회에는 포함하지 않는다(이미 닫힌 봉만 필요).

### 일 경계

일·주·월·4시간 경계는 거래소 관례를 따른다. 현재는 00:00 UTC(업비트·바이낸스 실제 경계, 한국시간 09시)를 적용한다. 빗썸(00:00 KST)의 경계 분기와 데이터 정리는 후속 과제로 둔다.

## 검증

| 항목 | 규칙 |
|------|------|
| 주기 | `1m`, `1h`, `4h`, `1d`, `1w`, `1M` 중 하나 |
| 조회 개수 | 1 이상 200 이하. 미지정 시 기본값 60 |

## 정렬

- 시간 오름차순(과거부터)으로 반환한다

## 빈 결과

- 해당 조건의 캔들이 없으면 빈 결과를 반환한다

## 커서

- 커서가 없으면 최신 캔들부터, 값이 있으면 해당 시각 이전 캔들을 조회한다
