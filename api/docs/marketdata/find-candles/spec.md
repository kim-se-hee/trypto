## 목적

캔들(OHLC) 차트 데이터를 조회한다. 시계열 캔들 데이터를 커서 기반 페이징으로 제공한다.

- 거래소별, 코인별 캔들 차트를 제공하기 위한 기능이다

### 사용처

- 마켓 탭에서 거래소·코인을 선택하면 캔들 차트를 표시한다

## 조회 항목

- 거래소·코인별 캔들(OHLC) 시계열 데이터: 각 캔들의 시가·고가·저가·종가와 시각
- 6개 주기(1분, 1시간, 4시간, 일, 주, 월)의 캔들
- 커서 기반 페이징으로 과거 캔들을 스크롤하며 조회
- 최신 윈도우 조회 시 아직 닫히지 않은 현재 구간의 캔들은 실시간으로 집계해 조회 결과에 포함

## 진행 중 캔들

캔들은 닫힌 뒤 틱을 집계하여 저장된다. 그래서 아직 안 끝난 봉(예: 오늘 진행 중인 일봉)은 저장소에 없다.

서버가 이 봉을 안 주면 클라이언트가 직접 만들어야 한다. 그런데 클라이언트는 페이지를 연 뒤부터의 가격만 받으므로 오늘 첫 시가를 모르는 등 정보가 부족할 수 있다.
결국 마켓 탭을 연 순간의 가격을 시가로 써서 봉 모양이 실제와 달라진다.

그래서 차트의 최신 윈도우를 조회하는 경우 서버가 진행 중인 최신 봉을 직접 계산해 조회 결과에 포함시킨다.

### 일 경계

일·주·월·4시간 봉이 새로 시작하는 시각은 거래소마다 다르다. 각 거래소의 기준에 맞춰 봉을 자른다.

| 거래소 | 하루 시작 |
|--------|----------|
| 업비트·바이낸스 | 한국시간 09시 (= 00:00 UTC) |
| 빗썸 | 한국시간 0시 (= 15:00 UTC) |

거래소마다 기준 시간대가 하나이고(위 표), 주봉·월봉·4시간봉도 모두 같은 시간대를 따른다.

- 주봉: 월요일 0시에 시작
- 월봉: 매월 1일 0시에 시작
- 4시간봉: 0시부터 4시간마다 시작 (0·4·8·12·16·20시)

여기서 "0시"는 그 거래소의 기준 시간대다. 예를 들어 빗썸은 전부 한국시간 0시, 업비트는 전부 UTC 0시(= 한국시간 9시)에 봉이 시작한다.

## 검증

| 항목 | 규칙 |
|------|------|
| 주기 | `1m`, `1h`, `4h`, `1d`, `1w`, `1M` 중 하나 |
| 조회 개수 | 1 이상 200 이하. 미지정 시 기본값 60 |

## 정렬

- 시간 오름차순(과거부터)으로 반환한다

## 빈 결과

- 해당 조건의 캔들이 없으면 빈 결과를 반환한다

## 커서

- 커서가 없으면 최신 캔들부터, 값이 있으면 해당 시각 이전 캔들을 조회한다
