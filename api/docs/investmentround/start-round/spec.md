## 목적

시드머니와 투자 원칙을 설정하여 새 투자 라운드를 시작한다.

- 모의투자 환경 초기화: 거래소별 시드머니, 긴급 자금 상한, 투자 원칙 설정
- 거래소별 지갑과 초기 잔고 자동 생성
- 투자 복기를 위한 원칙 설정 기반 마련

## 라운드 시작 조건

- 사용자당 진행 중인 라운드는 1개만 허용한다
- 이미 진행 중인 라운드가 있으면 새 라운드를 시작할 수 없다

## 시드머니 설정

- 시드머니는 원화 현금이며, 국내 원화 거래소(업비트)에만 입력한다
- 시드머니 제한: 최소 100만 KRW, 최대 5,000만 KRW
- 그 외 거래소(빗썸, 바이낸스)에는 시드머니를 직접 넣을 수 없다
  - 해당 거래소의 자금은 송금으로 마련한다
  - 특히 해외 거래소 자금은 국내에서 매수한 코인을 송금하여 마련하며, 이 과정 자체가 이 서비스가 제공하려는 체험이다
- 시드머니 총액은 원화 단일 통화로 집계한다

## 긴급 자금 투입 상한

- 라운드 시작 시 1회 투입 상한을 설정한다
- 상한은 원화 기준이며 최대 100만원 (0원 = 긴급 자금 미사용)
- 긴급 충전 가능 횟수: 최대 3회

## 투자 원칙 설정

- 각 원칙은 선택적으로 활성화/비활성화 가능하다
- 서버 검증: 비율은 0 초과, 횟수는 1 이상 정수만 확인한다

| 원칙 | 원칙 유형 | 설정값 의미 | 단위 |
|------|----------|------------|------|
| 손절 | LOSS_CUT | 보유 코인 손실률이 설정값에 도달하면 매도해야 함 | % |
| 익절 | PROFIT_TAKE | 보유 코인 수익률이 설정값에 도달하면 매도해야 함 | % |
| 추격 매수 금지 | CHASE_BUY_BAN | 일일 상승률이 설정값 이상인 코인 매수 금지 | % |
| 물타기 제한 | AVERAGING_DOWN_LIMIT | 코인별 물타기 최대 횟수 | 회 |
| 과매매 제한 | OVERTRADING_LIMIT | 일일 주문 최대 횟수 | 회 |

## 라운드 시작 시 초기화

- 진행 중 상태의 새 라운드를 생성한다 (회차: 이전 라운드 수 + 1)
- 활성화된 투자 원칙을 생성한다
- 거래소별 지갑을 모두 자동 생성한다
  - 업비트 지갑: 입력한 시드머니를 원화 가용 잔고로 설정한다
  - 그 외 거래소 지갑: 잔고 없이 생성한다 (송금을 받을 수 있도록 지갑 자체는 존재해야 한다)
