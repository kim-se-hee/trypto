## 목적

진행 중인 투자 라운드를 종료한다.

- 사용자가 라운드 종료 시점을 명시적으로 확정할 수 있게 한다.
- 종료 시점 이후 랭킹/복기 조회 기준을 고정한다.

## 라운드 상태

- 라운드는 진행 중, 파산, 종료 상태를 가진다.
- 라운드 시작 시점에는 진행 중 상태로만 만들어진다.

## 종료 데이터 보존 정책

- 종료는 라운드를 지우지 않고 종료 상태로 전환하는 방식으로 처리한다.
- 라운드를 종료 상태로 바꾸고 종료 시각을 남기는 것만 수행한다.
- 지갑/주문/복기 관련 이력 데이터는 삭제하지 않는다.

## 종료 권한과 상태 전이

- 본인 라운드만 종료할 수 있다.
- 진행 중인 라운드는 종료 상태로 전환하고 종료 시각을 서버 현재 시각으로 남긴다.
- 이미 종료된 라운드는 상태를 바꾸지 않고 기존 종료 시각 그대로 성공으로 응답한다(멱등 처리).
- 파산한 라운드는 종료할 수 없고, 진행 중이 아닌 라운드도 종료 대상이 아니다.
- 종료 후 활성 라운드 조회에서 해당 라운드는 더 이상 반환되지 않는다.
- 동시에 들어온 종료 요청은 하나만 성공하고 나머지는 충돌로 처리한다.
