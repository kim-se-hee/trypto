## 목적

로그인된 사용자를 로그아웃한다. 서버의 세션을 무효화하고 클라이언트의 인증 쿠키를 제거한다. 소셜 provider(카카오·구글 등) 종류와 무관하게 동일하게 동작한다.

## 처리 기준

- 유효한 세션이면 무효화하고, 세션 쿠키가 없거나 이미 죽은 세션이면 지울 것 없이 성공으로 처리한다.

## 세션 무효화 (서버 쪽 작업)

- 로그인 시 서버는 세션 ID를 만들어 Redis에 저장하고, 그 값을 `SESSION` 쿠키로 브라우저에 내려준다.
- 로그아웃은 요청 쿠키에 담겨 온 세션 ID를 Redis에서 삭제한다.
- 삭제 후에는 그 세션 ID를 다시 들고 와도 서버가 인증을 인정하지 않는다.

## 쿠키 제거 (브라우저 쪽 작업)

- 브라우저는 서버가 지우라고 하기 전까지 `SESSION` 쿠키를 계속 들고 있으므로, 응답으로 명시적으로 제거를 지시해야 한다.
- 같은 이름 `SESSION` 쿠키를 만료로 응답에 실어 보내 브라우저가 기존 쿠키를 덮어쓰고 버리게 한다.
