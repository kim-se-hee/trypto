## 소셜 신원과 회원 연결 처리

소셜 신원과 회원이 분리되어 있다.
탈퇴 시 소셜 신원의 연결을 해제한다. 즉 소셜 신원 테이블의 `user_id` 를 비운다. 이를 통해 그 계정으로는 즉시 로그인이 불가능해지고, 같은 소셜 계정이 나중에 새 회원과 연결될 수 있다. 
회원 행은 남기고 탈퇴 시각 기록 + 닉네임 익명 교체만 한다. 이를 통해 포트폴리오·랭킹 노출이 유지된다.

### 생애주기 예 : 가입 → 탈퇴 → 30일 후 재가입

| 시점 | social_identity | user |
|---|---|---|
| 가입 | id=7 · kakao/123 · user_id=42 | 42 · identity=7 · 코인왕 · deleted_at=NULL |
| 탈퇴(7/13) | id=7 · kakao/123 · user_id=NULL | 42 · identity=7 · 탈퇴한사용자77 · deleted_at=7/13 |
| 재가입(8/15) | id=7 · kakao/123 · user_id=99 | 42 행 그대로 + 99 · identity=7 · 무던한사용자88 · deleted_at=NULL |

- 소셜 신원 행(id=7)은 세대가 바뀌어도 그대로 남아 그 소셜 계정의 이력 앵커가 된다.
- 재가입 30일 판정: `user` 에서 `social_identity_id = 7` 이고 deleted_at 이 있는 행들의 가장 최근 탈퇴 시각을 본다.


## API 명세

### `DELETE /api/users/me`

본인 계정 탈퇴. 인증된 세션의 주인 회원을 탈퇴 처리한다. 요청 본문과 쿼리 파라미터는 없다.

- 성공: `200` + 세션 쿠키 만료(`Set-Cookie: SESSION=; Max-Age=0`, 로그아웃과 동일한 방식) + 아래 바디

```json
{
  "status": 200,
  "code": "SUCCESS",
  "message": "회원 탈퇴가 완료되었습니다.",
  "data": null
}
```

- 실패: 아래 에러 응답

## 에러 응답

| code | status | 설명 |
|------|--------|------|
| UNAUTHENTICATED | 401 | 세션이 없거나 유효하지 않음. 탈퇴 회원의 잔여 세션도 인증 단계에서 이 응답으로 거부된다 |
| USER_NOT_FOUND | 404 | 세션의 회원이 존재하지 않음 |
| USER_ALREADY_DELETED | 409 | 이미 탈퇴 처리된 회원 (신규 코드. 동시 탈퇴 요청 등으로 도달 가능) |
