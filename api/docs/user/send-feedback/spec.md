## 목적

회원이 서비스에 대한 사용 후기나 개선 요청을 운영자에게 전달한다. 
회원은 보내는 것으로 역할이 끝난다. 보낸 뒤 확인하거나 고치거나 답변을 받는 흐름은 이 기능에 없다.

## 용어 정의

- 피드백 = 작성자 + 본문 + 작성 시각.
- 사용 후기든 개선 요청이든 구분하지 않고 글 하나로 받는다. 회원이 무엇을 말하려는지는 본문이 스스로 드러내며, 분류는 운영자의 몫이다.
- 별점·만족도 점수 같은 정량 평가는 받지 않는다. 피드백은 글로만 이루어진다.

## 권한

- 로그인한 회원만 보낼 수 있다. 비로그인 사용자는 보낼 수 없다.
- 작성자는 언제나 로그인 세션의 주인이다. 다른 회원의 이름으로 보낼 수 없다.
- 탈퇴 회원은 로그인 세션을 가질 수 없으므로 보낼 수 없다.

## 검증

- 본문은 20자 이상 1000자 이하여야 한다.
- 공백만으로 채운 본문은 빈 본문으로 본다.

## 작성 횟수

- 횟수 제한을 두지 않는다. 같은 회원이 피드백을 몇 번이든 보낼 수 있다.
- 여러 번 보낸 피드백은 서로 덮어쓰지 않고 각각 별개의 피드백으로 보존된다. 나중에 보낸 것이 이전 것을 대체하지 않는다.

## 접수 이후

- 접수한 피드백은 보존하는 것으로 처리가 끝난다.
- 운영자 알림, 자동 응답, 처리 상태(검토중·반영·반려) 관리는 하지 않는다. 운영자가 필요할 때 직접 들여다보는 것을 전제로 한다.
- 회원에게는 접수되었다는 사실만 알린다.

## 회원과의 관계

- 피드백은 회원과 생명주기가 분리된다. 작성자가 탈퇴해도 이미 보낸 피드백은 지우지 않고 그대로 보존한다. 회원이 사라졌다는 이유로 개선 근거가 사라져서는 안 되기 때문이다.

## 이번 범위 밖

다음은 이번 기능에 포함하지 않는다. 필요해지면 별도 기능으로 다룬다.

- 회원이 자신이 보낸 피드백을 다시 조회하는 것
- 보낸 피드백의 수정·삭제
- 피드백의 공개 노출 (다른 회원에게 보여주는 것)
- 운영자 답변과 처리 상태 관리
- 도배 방지를 위한 작성 횟수 제한
