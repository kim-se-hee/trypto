## 목적

기간별(일간/주간/월간) 수익률 랭킹을 조회한다.


## 용어 정의

- 기간: 일간/주간/월간 중 선택
- 커서: 이전 페이지의 마지막 순위 값 (첫 페이지는 생략)
- 페이지 크기: 1~50, 기본값 20

## 조회 항목

- 순위
- 사용자 식별 정보와 닉네임
- 수익률
- 거래 횟수
- 포트폴리오 공개 여부
- 커서 페이지 정보: 다음 페이지 커서와 다음 페이지 존재 여부

## 기간 검증

| 항목 | 규칙 |
|------|------|
| 기간 | `DAILY`, `WEEKLY`, `MONTHLY` 중 하나여야 한다 |

## 기준 날짜 결정

- 기준 날짜 파라미터가 없으면 해당 기간의 **최신 집계 날짜**를 사용한다
- 기준 날짜 파라미터가 있으면 해당 날짜의 집계 결과를 조회한다

## 동률 처리

순위를 부여할 때 아래 기준으로 정렬하여 고유한 순위를 할당한다.

1. 수익률이 높은 유저가 상위
2. 수익률이 동일하면 거래 횟수가 적은 유저가 상위 (적은 매매로 같은 수익 = 더 효율적)
3. 거래 횟수도 동일하면 라운드 시작이 빠른 유저가 상위
4. 위 기준이 모두 동일하면 사용자 ID 순
