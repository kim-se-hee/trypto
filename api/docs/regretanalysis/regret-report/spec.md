## 목적

투자 라운드의 복기 리포트를 조회한다.

- 사용자가 설정한 투자 원칙을 얼마나 지켰는지, 위반이 자산에 어떤 영향을 미쳤는지 정량적으로 분석한다
- "원칙을 지켰다면?" 시뮬레이션을 통해 규칙별 수익률 영향(impactGap)을 보여준다
- 놓친 수익 금액(missedProfit)으로 원칙 준수의 가치를 체감하게 한다
- 위반 거래별로 어떤 규칙을 위반했는지, 결과적으로 이익/손실이었는지 보여준다

## 조회 항목

요청한 거래소의 다음 데이터를 제공한다.

- 요약 지표: 놓친 수익, 실제 수익률, 원칙 준수 시 수익률, 총 위반 횟수
- 규칙별 시나리오(Rule Impact): 각 원칙을 지켰다면 달라졌을 수익률 시뮬레이션
- 위반 거래 목록(Violation Detail): 원칙을 위반한 개별 거래와 위반 규칙, 손익

거래소마다 기축통화가 다르므로(국내: KRW, 바이낸스: USDT) 거래소별로 요청한다.

> 복기 분석 공통 계산 규칙(원칙별 손실 금액 계산, 기준가 결정 로직, 공통 용어)은 [../business-rules.md](../business-rules.md)를 참조한다. 아래는 이 기능 고유의 규칙이다.

## 조회 조건

- 해당 투자 라운드의 복기 리포트를 조회한다
- 라운드 소유자만 조회할 수 있다 (본인 라운드 검증)
- 요청한 거래소의 지갑이 해당 라운드에 존재해야 한다. 존재하지 않으면 조회할 수 없다
- 라운드 상태는 ACTIVE 또는 ENDED 모두 조회 가능하다

## 리포트 생성

배치에서 리포트를 사전 생성한다. API는 배치가 생성한 데이터를 조회만 한다. 리포트가 아직 없으면 (라운드 시작 당일 등) 조회할 수 없다.

## 손실 계산

원칙별 손실 금액 계산과 기준가 결정 로직은 [../business-rules.md](../business-rules.md)를 참조한다.

**위반분 우선 매칭 (매수 위반 규칙):**

`CHASE_BUY_BAN`, `AVERAGING_DOWN_LIMIT`, `OVERTRADING_LIMIT`(매수)의 경우, 위반 매수분이 먼저 매도된 것으로 간주하여 기준가를 결정한다. 이를 위해 위반 주문뿐 아니라 **해당 코인의 이후 매도 이력**도 함께 조회한다. 상세 로직은 [../business-rules.md의 기준가 결정 로직](../business-rules.md#기준가-결정-로직)을 참조한다.

## 시나리오 생성

거래소별로 시나리오를 생성한다.

1. 해당 거래소에서 해당 원칙의 모든 위반 거래의 위반 손익을 합산하여 총 위반 손익(totalLossAmount)을 산출한다
2. 원칙 준수 시 총 자산 = 실제 총 자산 + 총 위반 손익
3. 원칙 준수 시 수익률 = (원칙 준수 시 총 자산 - 총 투입금) / 총 투입금 × 100
4. 수익률 영향 차이(impactGap) = 원칙 준수 시 수익률 - 실제 수익률 (%p)

**실제 총 자산 기준:** 배치가 적재한 마지막 일별 자산 스냅샷의 총 자산을 사용한다.

## 놓친 수익 계산

규칙을 지켰다면 얼마나 더 벌 수 있었는지를 거래소별로 각각 산출한다. 거래소마다 기축통화가 다르므로(국내: KRW, 바이낸스: USDT) 합산하지 않고 따로 보여준다. 환율 변동이 손익에 섞이는 것을 방지하기 위함이다.

```
missedProfit = max(0, Σ all loss_amounts)
```

- 해당 거래소의 모든 위반 거래 위반 손익을 합산한다
- 합산 결과가 음수면 0으로 처리한다 (원칙을 지켰어도 손해인 경우 "놓친 수익 없음")

## 갱신 정책

- **ACTIVE 라운드**: 배치가 매일 갱신한다. API는 배치 결과를 조회만 한다. 미실현분은 배치 시점의 현재가로 계산되어 있으며, 다음 배치까지 변하지 않는다
- **ENDED 라운드**: 종료 시 1회 확정 생성 후 변하지 않는다. 실현분은 매도 체결가로 고정된다
- **리포트 미존재 시** (라운드 시작 당일 등): 조회할 수 없다. 배치 실행 후 조회 가능하다

## 규칙 기준값 단위

규칙 유형에 따라 기준값의 단위가 결정된다.

| 규칙 유형 | thresholdUnit | 예시 |
|-----------|---------------|------|
| `LOSS_CUT` | `%` | 10% 하락 시 손절 |
| `PROFIT_TAKE` | `%` | 30% 상승 시 익절 |
| `CHASE_BUY_BAN` | `%` | 20% 이상 급등 시 매수 금지 |
| `AVERAGING_DOWN_LIMIT` | `회` | 최대 2회까지 물타기 허용 |
| `OVERTRADING_LIMIT` | `회` | 하루 최대 10회 거래 |

## 위반 거래 그룹핑

- **주문 시점 위반**: 같은 주문에서 발생한 위반 기록을 하나의 거래로 그룹핑한다. 하나의 주문이 여러 규칙을 위반한 경우 위반한 규칙 목록으로 묶는다
- **가격 모니터링 위반**: 각 위반이 독립적인 거래 항목이 된다 (손절/익절 미이행)
- 거래 손익은 해당 거래의 전체 실현/미실현 손익이다 (규칙별 위반 손익 합산이 아닌, 거래 자체의 손익)

## 위반 거래 구분

- 위반 거래는 거래 손익 부호로 전체/손실/수익으로 구분된다. 서버는 별도로 필터링하지 않고 구분 기준만 정의한다.
  - 전체: 모든 위반 거래
  - 손실: 거래 손익이 음수인 거래
  - 수익: 거래 손익이 0 이상인 거래

## 투자 원칙 없는 라운드

투자 원칙이 없는 라운드는 에러 대신 빈 규칙별 시나리오와 위반 거래 목록을 반환한다.
