## 목적

투자 라운드의 복기 그래프 데이터를 조회한다. 거래소별 일별 자산 추이를 시각화하여 실제 투자 성과를 보여주고, "원칙을 지켰다면?" 시뮬레이션과 BTC 홀드 벤치마크를 겹쳐 단순 보유 대비 성과를 비교하게 하며, 위반 발생 시점을 그래프 위에 표시한다.

## 용어 정의

요청한 거래소의 데이터를 제공한다. 거래소마다 기축통화가 다르므로(국내: KRW, 바이낸스: USDT) 거래소별로 요청한다.

1. **자산 추이 (Asset History)**: 일별 실제 자산(`actualAsset`), 규칙 준수 시 시뮬레이션 자산(`ruleFollowedAsset`), BTC 홀드 벤치마크(`btcHoldAsset`)
2. **위반 지점 마커 (Violation Markers)**: 위반 거래가 발생한 날짜와 해당 시점의 자산값

복기 분석 공통 용어(`expected_amount`, `actual_amount`, `loss_amount`, `기준가`)는 [../business-rules.md](../business-rules.md) 를 참조한다.

## 조회 항목

요청한 거래소에 대해 다음을 반환한다.

- 거래소별 일별 자산 추이: 실제 자산(`actualAsset`), 규칙 준수 시 시뮬레이션 자산(`ruleFollowedAsset`, 분홍 점선), BTC 홀드 벤치마크(`btcHoldAsset`, 주황 실선)
- 위반 지점 마커(빨간 점): 위반이 발생한 날짜와 해당 시점의 자산값

## 조회 조건

- 해당 투자 라운드의 그래프 데이터를 조회한다
- 라운드 소유자만 조회할 수 있다 (본인 라운드 검증)
- 일별 자산 스냅샷이 아직 없으면 (라운드 시작 당일 등) 오류가 아니라 빈 그래프를 반환한다. 자산 추이와 위반 마커는 빈 목록, 총 일수는 0이다
- 복기 리포트가 아직 없으면 위반이 없는 것으로 간주한다. 시뮬레이션 자산은 실제 자산과 같고, 위반 마커는 표시되지 않는다

## 실제 자산 추이

해당 라운드·거래소의 일별 총 자산을 사용한다.

## 규칙 준수 시 시뮬레이션 자산 계산

위반이 없었다면의 가상 자산 추이를 일별로 계산한다.

```
ruleFollowedAsset[day] = actualAsset[day] + Σ(위반 손실, 해당일까지 누적)
```

근사 계산이다. 완전한 포트폴리오 재시뮬레이션 대신 누적 손실 가산 방식을 사용한다. 실제로는 손실이 복리로 영향을 미치지만, 계산 복잡도 대비 직관적인 근사치를 제공한다.

위반 거래의 위반 손익 자체를 산출하는 원칙별 계산 규칙과 기준가 결정 로직은 [../business-rules.md](../business-rules.md) 를 참조한다.

**계산 절차:**

1. 복기 리포트의 위반 거래에서 해당 거래소의 위반 손익과 위반 발생 시각을 가져온다
2. 각 스냅샷 날짜에 대해, 해당일 이전(포함)에 발생한 위반 거래의 위반 손익을 누적 합산한다
3. `ruleFollowedAsset[day] = actualAsset[day] + cumulativeLoss[day]`

## BTC 홀드 벤치마크 계산

해당 거래소의 시드머니 전액으로 BTC만 매수하여 보유했을 때의 가상 자산 추이를 계산한다.

```
btcQuantity = seedMoney / btcPriceAtRoundStart
btcHoldAsset[day] = btcQuantity × btcPriceAtDay
```

- 국내 거래소: KRW 마켓 BTC 종가 기준
- 바이낸스: USDT 마켓 BTC 종가 기준
- 수수료는 적용하지 않는다 (벤치마크 비교 목적)

**계산 절차:**

1. 해당 거래소의 시드머니와 라운드 시작일의 BTC 가격을 조회한다
2. `btcQuantity = seedMoney / btcPriceAtRoundStart`
3. 각 스냅샷 날짜의 BTC 가격을 조회하여 `btcHoldAsset`을 계산한다

**BTC 가격 데이터 소스:**
- 저장된 BTC 시세 이력에서 일별 종가를 조회한다

## 위반 마커 생성

복기 리포트의 위반 거래 목록을 기반으로 그래프 위 마커 데이터를 생성한다.

1. 해당 거래소의 위반 거래에서 위반 발생 시각의 날짜를 스냅샷 날짜에 매핑한다
2. 해당 스냅샷 날짜의 실제 자산을 마커의 자산값으로 설정한다
3. 같은 날짜에 여러 위반이 있으면 **날짜당 하나로 합친다**. 세부 내용은 리포트 API의 위반 거래 목록에서 확인한다

## 규칙 토글 시뮬레이션

- 서버는 전체 규칙을 준수했을 때의 시뮬레이션 자산만 제공한다. 개별 규칙을 켜고 끄는 조합별 자산은 미리 계산하지 않는다.
- 개별 규칙 토글 시의 라인 변화는 클라이언트가 근사로 보여주며, 정확한 재시뮬레이션이 아니라 직관적 시각 피드백이 목적이다.
