## 목적

거래소 지갑의 코인 입금 주소를 조회한다.

- 사용자가 송금 전 도착 거래소의 입금 주소를 확인하고 복사할 수 있도록 한다

## 조회 항목

- 입금 주소 — (지갑, 코인) 단위로 고유한 입금 주소 문자열
- 최초 조회 시 생성하고 이후 동일 주소를 반환한다

KRW(법정화폐)는 송금할 수 없고 USDT는 암호화폐이므로 송금 가능하다는 공통 규칙은 [../business-rules.md](../business-rules.md) 를 따른다. 아래는 입금 주소 발급 고유 규칙이다.

## 입금 주소 생성

- 입금 주소는 없으면 새로 발급하고, 이미 있으면 기존 주소를 돌려준다
- 네트워크(체인)와 태그는 사용하지 않는다 (시뮬레이션 단순화)

## 입금 제한

- KRW는 암호화폐가 아니므로 입금 주소가 없다
- 국내 거래소(업비트, 빗썸)의 기축통화(KRW)는 입금 주소 발급이 차단된다
- 바이낸스의 기축통화(USDT)는 암호화폐이므로 입금 주소 발급이 가능하다
