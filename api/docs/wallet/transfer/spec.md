## 목적

같은 라운드에 속한 서로 다른 거래소 지갑 사이에서 코인을 이동한다.

- 사용자가 한 거래소 지갑의 코인을 같은 라운드의 다른 거래소 지갑으로 옮길 수 있도록 한다

## 이동 대상

- 이동 대상은 거래소에 상장된 코인에 한한다.
- 기준 통화(KRW·USDT)는 상장 코인이 아니므로 이동할 수 없다. 아래 상장 확인 단계에서 거부된다.

## 검증 단계

검증에 실패하면 송금이 생성되지 않고 에러를 반환한다.

1. **지갑 존재 확인**: 출발 지갑과 도착 지갑이 모두 존재하는지 확인한다
2. **소유 확인**: 출발 지갑이 요청자의 것인지 확인한다
3. **동일 라운드 확인**: 출발 지갑과 도착 지갑이 같은 라운드에 속하는지 확인한다
4. **서로 다른 지갑 확인**: 출발 지갑과 도착 지갑이 서로 다른지 확인한다
5. **도착 거래소 상장 확인**: 도착 지갑의 거래소가 해당 코인을 취급하는지 확인한다
6. **잔고 검증**: 출발 지갑의 가용 잔고 ≥ 송금 수량

### 도착 거래소 상장 확인

거래소마다 상장 코인이 다르므로, 도착 거래소가 취급하지 않는 코인은 옮길 수 없다.

- 도착 지갑의 거래소에 해당 코인이 상장되어 있지 않으면 거부한다.
- 출발 거래소는 별도로 검증하지 않는다. 미상장 코인은 출발 지갑에 잔고 자체가 없으므로 잔고 검증에서 걸러진다.
- 이 확인은 **잔고 조회보다 먼저** 수행한다. 순서가 뒤바뀌면 미상장 코인 요청이 잔고 부재로 먼저 실패하여, 거절 사유가 "취급하지 않는 코인"이 아니라 "잔고 없음"으로 잘못 보고된다.

## 잔고 변동

| 결과 | 출발 지갑 | 도착 지갑 |
|------|----------|----------|
| SUCCESS | 가용 잔고에서 송금 수량 차감 | 가용 잔고에 송금 수량 추가 |

- 송금은 검증을 모두 통과하면 항상 SUCCESS로 처리된다.
