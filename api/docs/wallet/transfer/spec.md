## 목적

같은 라운드에 속한 서로 다른 거래소 지갑 사이에서 코인을 이동한다.

- 사용자가 한 거래소 지갑의 코인을 같은 라운드의 다른 거래소 지갑으로 옮길 수 있도록 한다

## 검증 단계

검증에 실패하면 송금이 생성되지 않고 에러를 반환한다.

1. **멱등성 검사**: 요청자가 보낸 멱등성 키로 기존 송금을 조회한다. 존재하면 기존 결과를 반환한다
2. **지갑 존재 확인**: 출발 지갑과 도착 지갑이 모두 존재하는지 확인한다
3. **동일 라운드 확인**: 출발 지갑과 도착 지갑이 같은 라운드에 속하는지 확인한다
4. **잔고 검증**: 출발 지갑의 가용 잔고 ≥ 송금 수량
5. **서로 다른 지갑 확인**: 출발 지갑과 도착 지갑이 서로 다른지 확인한다

## 잔고 변동

| 결과 | 출발 지갑 | 도착 지갑 |
|------|----------|----------|
| SUCCESS | 가용 잔고에서 송금 수량 차감 | 가용 잔고에 송금 수량 추가 |

- 송금은 검증을 모두 통과하면 항상 SUCCESS로 처리된다.
