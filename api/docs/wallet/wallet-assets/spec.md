## 목적

지갑의 잔고 목록을 조회하는 REST API다.

- 입출금 탭에서 사용자의 보유 잔고를 제공한다

## 조회 항목

- 사용자가 보유한 코인의 available(사용 가능) 잔고와 locked(잠금) 잔고
- 기축통화 잔고는 별도 필드로 분리하여 제공한다
- 현재가는 실시간 시세로 따로 받으므로 이 응답에 포함하지 않는다

잔고 처리 공통 규칙은 [../business-rules.md](../business-rules.md) 를 참조한다. 아래는 이 기능 고유의 규칙이다.

## 검증

| 항목 | 규칙 |
|------|------|
| 지갑 존재 | 조회하려는 지갑이 존재해야 한다 |
| 지갑 소유권 | 지갑이 속한 라운드의 소유자가 요청자 본인인지 검증한다 |
