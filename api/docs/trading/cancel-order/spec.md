## 목적

사용자가 아직 체결되지 않은(미체결) 지정가 주문을 철회할 수 있게 한다.

## 용어 정의

- 미체결 주문: 지정가로 생성되어 아직 체결되지 않은 상태의 주문. 취소 가능한 유일한 상태다.
- 점유 잔고 복원: 주문 생성 시 잠긴 잔고를 사용 가능 잔고로 되돌리는 것.

## 비지니스 규칙

### 취소 가능 대상

- 미체결 상태의 주문만 취소할 수 있다. 이미 체결·취소·실패한 주문은 취소할 수 없다.

### 소유권 검증

- 취소 요청에 지갑을 함께 받아 주문의 지갑과 일치하는지 검증한다.
- 불일치 시 주문 없음으로 처리한다. 주문의 존재 여부를 노출하지 않기 위해 권한 거부가 아니라 없음으로 응답한다.

### 멱등성

- 이미 취소된 주문에 대한 취소 요청은 예외 없이 현재 상태를 그대로 반환한다. 네트워크 재시도나 중복 요청에 안전하다.

### 상태 전이와 잔고 복원

- 취소가 확정되면 주문을 미체결에서 취소 상태로 전이한다. 전이는 원자적으로 일어나며, 성공한 경우에만 취소가 진행된다.
- 전이 성공 시 주문 생성 때 점유한 잔고를 즉시 사용 가능 잔고로 복원한다.
  - 지정가 매수: 기준 통화(KRW/USDT)를 점유했던 만큼(체결금액 + 수수료) 해제한다.
  - 지정가 매도: 코인을 점유했던 만큼(체결 수량) 해제한다.
- 상태 전이와 잔고 해제는 하나의 작업 단위로 함께 이뤄진다. 주문은 취소됐는데 잔고는 점유된 상태로 남는 불일치를 막는다.

### 취소와 체결의 경합

- 사용자가 체결 직전에 취소 버튼을 누르면 취소 요청과 엔진의 체결 확정이 같은 주문을 동시에 노린다.
- 양쪽 모두 미체결 상태를 전제로 상태를 바꾸므로 둘 중 하나만 성공한다.
  - 체결이 먼저 확정되면 취소는 실패하고 취소 불가로 응답한다.
  - 취소가 먼저 확정되면 엔진의 체결 후속 처리(잔고·알림·보유 코인)가 모두 건너뛰어진다.
