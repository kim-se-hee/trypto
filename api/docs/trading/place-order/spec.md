## 목적

업비트·빗썸·바이낸스 같은 거래소의 시장가/지정가 주문을 생성한다.

## 용어 정의

- 잔고(Balance): 코인별로 `available`(사용 가능)과 `locked`(잠금) 두 값으로 구성된다. `available`은 주문·송금에 쓸 수 있는 금액이고, `locked`는 미체결 지정가 주문 점유나 송금 동결 등으로 사용 불가 처리된 금액이다.
- 보유 코인(Holding): 코인별 평균 매수가와 보유 수량. 평균 매수가 정밀도 보장을 위한 누적 매수 금액과, 물타기 횟수를 함께 추적한다.
- 기준 통화: 국내 거래소는 KRW, 해외 거래소(바이낸스)는 USDT.

## 비지니스 규칙

### 잔고 모델

- 사용 불가 처리(지정가 점유, 송금 동결 등)는 `available`을 줄이고 `locked`를 같은 크기로 늘린다.
- 잔고 변동은 차감, 증가, 점유, 점유 해제 네 가지 연산으로 구성된다.

### 보유 코인 갱신

- 매수 체결 시 평균 매수가 = (기존평균 × 기존수량 + 체결가 × 체결수량) / (기존수량 + 체결수량).
- 매도 체결은 평균 매수가를 바꾸지 않고 수량만 줄인다. 전량 매도 시 평균 매수가를 0으로 리셋한다.
- 평균 매수가 정밀도를 위해 누적 매수 금액을 별도로 관리한다.
- 물타기(손실 중 추가 매수) 횟수는 코인별로 라운드 전체 기간 동안 누적한다.
- 갱신 시점: 시장가 주문은 즉시 체결이므로 주문 생성 시점에 갱신한다. 지정가 주문은 체결 매칭 시점에 갱신한다(매칭 흐름 참조).

### 주문 입력

- 매수는 주문 금액(기준 통화)을 입력한다. 지정가 매수는 수량 입력도 허용한다.
- 매도는 주문 수량(코인)을 입력한다. 지정가 매도는 금액 입력도 허용한다.

### 주문 금액 제한

| 구분 | 최소 주문 금액 | 최대 주문 금액 |
|------|-----------|----------|
| 국내 거래소 | 5,000 KRW | 10억 KRW |
| 해외 거래소(바이낸스) | 5 USDT | 제한 없음 |

### 체결 계산

- 코인 체결 수량은 소수점 8자리까지 표현하고 그 이하는 버림한다.
- 시장가 매수 체결 수량 = floor(주문금액 / 현재가, 8자리). 지정가 매수 체결 수량 = floor(주문금액 / 지정가, 8자리). 시장가는 체결가가 현재가이고 지정가는 체결가가 지정가이기 때문이다.
- 매도는 입력한 주문 수량이 그대로 체결 수량이 된다(8자리 초과분은 버림).
- 매수 시 버림으로 체결되지 못한 잔여 금액(주문금액 − 실제 체결 금액)은 기준 통화 잔고에 그대로 남는다.
- 실제 체결 금액 = 체결 수량 × 체결가.
- 수수료 = 실제 체결 기준 통화 금액 × 수수료율. 수수료율은 거래소별 고정 요율이다.

### 매수/매도 가능 판정

- 주문 가능 금액 = 기준 통화 잔고 − 송금 예정 금액 − 동결 금액 − 미체결 지정가 매수 주문 점유 금액. 매수는 실제 체결 금액 + 수수료 ≤ 주문 가능 금액이어야 한다.
- 주문 가능 수량 = 보유 수량 − 송금 예정 수량 − 동결 수량 − 미체결 지정가 매도 주문 점유 수량. 매도는 체결 수량 ≤ 주문 가능 수량이어야 한다.

### 시장가 주문

- 부분 체결 없이 전량 즉시 체결한다.
- 체결가는 현재가다. 매수·매도 모두 가능 조건을 만족하면 현재가로 즉시 전량 체결한다.

### 지정가 주문

- 부분 체결 없이 가격 조건 달성 시 전량 체결한다. 체결가는 지정가다.
- 즉시 체결되지 않으므로, 체결 대기 중 잔고 이중 사용을 막기 위해 주문 생성 시점에 잔고를 점유한다. 점유된 금액·수량은 사용할 수 없다.
  - 매수: 실제 체결 금액 + 수수료를 기준 통화 잔고에서 점유한다.
  - 매도: 체결 수량을 코인 보유 수량에서 점유한다.
- 체결 조건: 매수는 현재가 ≤ 지정가, 매도는 현재가 ≥ 지정가.
- 생성 시 미체결 상태로 저장한다. 실제 체결 판정은 매칭 흐름이 담당한다(pending-order-matching 참조).

### 멱등성

- 클라이언트가 멱등성 키를 생성해 함께 보낸다. 동일 키로 중복 요청이 오면 새 주문을 만들지 않고 기존 주문 결과를 반환한다.

### 투자 원칙 위반 판정

주문 생성 시 투자 원칙 위반 여부를 함께 판정해 투자 복기용 그래프 데이터를 남긴다. 위반해도 주문은 정상 진행되며 위반 기록만 저장한다.

주문 시점에 검사하는 규칙:

- 추격 매수 금지: 매수 주문 한정. 해당 코인 상승률이 설정값 이상이면 위반. 상승률은 데이터 소스에서 계산된 값(업비트·빗썸은 전일종가 대비, 바이낸스는 24시간 전 대비)을 조회한다.
- 물타기 제한: 매수 주문 한정. 평균 매수가 > 현재가(손실 중)인 코인을 추가 매수하면 물타기 1회. 코인별·라운드 전체 기간 누적 횟수가 설정값 이상이면 위반.
- 과매매 제한: 매수·매도 모두. 오늘(KST 00:00~23:59) 주문 횟수(미체결 지정가 포함)가 설정값 이상이면 위반.

가격 모니터링 규칙(손절·익절)은 주문 시점이 아니라 시간 기반 모니터링에서 별도 처리하므로 여기서 검사하지 않는다.

위반 시 위반 사유 상세(예: "상승률 5.2% ≥ 3.0%")를 주문과 묶어 기록한다.

### 권한·소유

- 주문은 요청에 담긴 지갑에 귀속된다.
