기존 스펙에 사용자 피드백을 반영한다. 피드백은 자유 텍스트로 들어오며, BR 추가/수정/삭제, 마커 해소 등이 일어날 수 있다.

## refine 전용 규칙

### `[CONFLICT]` 마커

피드백이 다음 두 경우에 해당하면 `[CONFLICT]` 로 보고 사용자에게 보고한다.

1. 같은 prefix 의 다른 스펙 md 와의 충돌 (또는 contracts 와의 충돌)
2. 같은 문서 내 변경 대상이 **아닌** 다른 BR 과의 충돌

사용자가 명시적으로 변경을 요청한 BR 자체와 피드백의 차이는 충돌이 아니다 — 그건 그냥 변경이다.

### BR 재번호

매 refine 마다 전체 BR 을 1번부터 순차 재번호한다. 결번을 두지 않는다.

### 계획 md 동기화

`docs/<prefix>/<feature>/plan.md` 가 존재하면 BR 번호 참조를 자동 갱신한다. 없으면 스킵한다.

### 변경 범위 제한

피드백이 다루지 않은 부분은 임의로 수정하지 않는다. 문장 다듬기·BR 재정렬·관련 없는 마커 해소 등 사용자가 명시적으로 요구하지 않은 변경은 금지한다. 재번호로 인해 번호가 바뀌는 것은 예외다.

## 흐름

### 1. 대상 형식 감지

`docs/<prefix>/<feature>/spec.md` 가 존재하면 그대로 진행한다.

`docs/<prefix>/<feature>.md` (단일 파일에 spec/plan 이 섞여 있던 이전 컨벤션) 가 감지되면 사용자에게 알리고 종료한다. refine 은 현재 컨벤션의 폴더 구조에서만 동작한다.

### 2. 피드백 수신

사용자가 자유 텍스트로 제공한 피드백을 받는다. 피드백 항목별로 의도를 식별한다:

- 신규 BR 추가
- 기존 BR 수정 (의미 유지)
- 기존 BR 삭제
- 마커 해소 (`[BLOCK]` / `[ASSUME]` 답변)
- 목적 / 추측 근거 섹션 수정

### 3. 변경 대상 식별

각 피드백 항목이 가리키는 BR 번호 또는 섹션을 결정한다. 사용자가 명시적으로 BR 번호를 지목하지 않은 경우 본문에서 의미가 가까운 항목을 매칭한다. 매칭이 모호하면 사용자에게 확인을 요청한다.

### 4. 충돌 검사

각 변경 항목에 대해 다음 두 종류의 충돌을 검사한다.

- 같은 prefix 의 다른 스펙 md 와 의미 충돌
- 같은 문서 내에서 변경 대상이 **아닌** 다른 BR 과 의미 충돌

변경 대상 자체와 피드백의 차이는 충돌이 아니다. 사용자가 그걸 바꿔달라고 한 것이다.

### 5. CONFLICT 처리

충돌이 발견되면 사용자에게 양쪽 안을 제시하고 선택을 받는다.

```
[CONFLICT] BR<n> 충돌

안 1: <기존 내용>
안 2: <피드백을 반영한 내용>
```

사용자가 한쪽을 선택하면 그 안을 채택한다. 충돌 항목이 여러 개면 각각 처리한다.

### 6. 적용 + 재번호

피드백을 반영해 BR 목록을 갱신한다.

- 신규 BR: 목록 끝에 추가
- 기존 BR 수정: 자리 유지하고 본문 갱신
- 기존 BR 삭제: 목록에서 제거

전체 BR 을 1번부터 순차 재번호한다. 결번을 두지 않는다.

### 7. 마커·추측 근거 정리

해소된 마커는 본문에서 제거한다. 새로 발생한 추측은 마커를 부착한다.

추측 근거 섹션을 갱신한다:
- 해소된 BR 번호의 항목 제거
- 새 추측이 있으면 추가
- 재번호로 인해 번호가 바뀐 항목은 새 번호로 갱신

### 8. 계획 md 동기화

`docs/<prefix>/<feature>/plan.md` 가 존재하면 BR 번호 참조를 자동 갱신한다.

재번호 매핑 (예: 변경 전 BR3 → 변경 후 BR2) 을 만들어 plan.md 안의 `[<번호>]` 형태 참조를 일괄 치환한다.

신규 BR 은 plan.md 에 자동으로 매핑되지 않는다. 변경 보고에 신규 BR 목록을 포함하여 사용자가 plan.md 의 어느 도메인 모델에 매핑할지 결정하게 한다.

plan.md 가 없으면 이 단계를 스킵한다.

### 9. 기능 index.md 갱신

필요 시 `docs/<prefix>/<feature>/index.md` 의 산출물 한 줄 요약을 갱신한다.

`docs/<prefix>/index.md` 의 기능 한 줄 요약도 변경되었으면 갱신한다.

### 10. 변경 보고

사용자에게 다음을 보고한다:

```
스펙 갱신 완료: docs/<prefix>/<feature>/spec.md

번호 매핑 (재번호):
- 변경 전 BR3 → 변경 후 BR2
- 변경 전 BR5 → 변경 후 BR4
...

신규 BR (계획 md 매핑 필요):
- BR<n>: <본문>

미해결 마커:
- [BLOCK] BR<n>: <본문>
- [ASSUME] BR<n>: <본문>

[CONFLICT] 처리 결과:
- BR<n>: 안 <1|2> 채택

계획 md 동기화:
- 자동 갱신 완료 / 대상 없음
```

해당 항목이 없으면 그 섹션은 생략한다.
