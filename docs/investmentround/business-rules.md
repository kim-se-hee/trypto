# 투자 라운드

- 라운드 시작 시 시드머니와 투자 원칙을 설정해야 한다
- 라운드 진행 중 긴급 자금 투입은 최대 3회까지 가능하다
- 1회 긴급 자금 투입 상한선은 라운드 시작 전에 설정한다

# 투자 원칙

- 라운드 시작 전에 나만의 투자 원칙을 설정해야 한다
- 손절 규칙: 설정한 손실률에 도달하면 반드시 매도해야 한다
- 익절 규칙: 설정한 수익률에 도달하면 반드시 매도해야 한다
- 추격 매수 금지: 설정한 비율 이상 상승한 코인은 매수하지 않아야 한다
- 물타기 제한: 같은 코인에 대한 물타기 횟수를 설정한 횟수 이내로 제한해야 한다
- 과매매 제한: 하루 매매 횟수를 설정한 횟수 이내로 제한해야 한다

## 원칙 체크 시점 분류

| 원칙 | 체크 시점 | 대상 |
|------|----------|------|
| 손절 | 가격 모니터링 (별도 스케줄러) | 보유 코인 |
| 익절 | 가격 모니터링 (별도 스케줄러) | 보유 코인 |
| 추격 매수 금지 | 주문 생성 시 | 매수 주문 |
| 물타기 제한 | 주문 생성 시 | 매수 주문 |
| 과매매 제한 | 주문 생성 시 | 매수/매도 모든 주문 |

## 주문 시 체크 원칙 상세

### 추격 매수 금지

- 매수 주문 시, 해당 코인의 상승률이 설정한 기준값 이상이면 위반
- 상승률 기준:
  - 업비트/빗썸: 전일종가 대비 상승률
  - 바이낸스: 24시간 전 대비 상승률
- 상승률은 웹소켓으로 수신되어 저장된 값을 조회한다 (계산은 데이터 소스에서 처리)
- 위반 예시: 설정값 3%, 현재 상승률 5.2% → 위반

### 물타기 제한

- 매수 주문 시, 평균 매수가 > 현재가(손실 중)인 코인을 추가 매수하면 물타기 1회
- 코인별로 라운드 전체 기간에 걸쳐 카운트한다
- Holding 엔티티의 `averaging_down_count` 필드로 관리
- 물타기 횟수(현재 포함) ≥ 설정값이면 위반
- 위반 예시: 설정값 3회, 현재까지 물타기 2회 + 이번 1회 = 3회 → 위반

### 과매매 제한

- 매수/매도 모든 주문 시 체크
- 오늘(KST 00:00~23:59) 주문 횟수를 카운트
- 미체결 지정가 주문도 카운트에 포함
- 오늘 주문 횟수 ≥ 설정값이면 위반
- 위반 예시: 설정값 10회, 오늘 기존 9회 + 이번 1회 = 10회 → 위반

## 위반 시 처리

- 위반해도 주문은 정상적으로 진행된다
- `RULE_VIOLATION` 테이블에 위반 기록을 저장한다
- 위반 기록은 투자 복기용 후회 그래프 분석에 활용된다
